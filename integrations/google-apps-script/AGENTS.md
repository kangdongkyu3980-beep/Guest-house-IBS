# AGENTS.md

## 적용 범위

이 문서는 `integrations/google-apps-script/` 아래 백엔드 연동 코드에 적용한다.

루트 `AGENTS.md`의 공통 규칙을 따른다.  
여기서는 Apps Script, Google Sheets 연동, 운영 데이터 변경에 대한 추가 기준을 둔다.

## 백엔드 작업 원칙

1. 승인 없는 운영 데이터 구조 변경은 피한다.
2. 저장 로직보다 검증 가능한 읽기 경로를 먼저 본다.
3. 부분 성공보다 일관된 실패 보고가 낫다.
4. 프론트엔드가 기대하는 응답 계약을 조용히 깨지 않는다.

## 백엔드 검증 규칙

1. 읽기와 쓰기 계약이 서로 맞는지 확인한다.
2. 저장 후 재조회 결과가 기대값과 일치하는지 확인한다.
3. 삭제, 보관, 복구 흐름이 섞이지 않는지 본다.
4. 시트 컬럼 순서나 이름에 의존하는 부분을 바꿀 때는 영향 범위를 먼저 적는다.
5. 실패 응답이 프론트엔드에서 식별 가능한 형태인지 확인한다.

## 큰 변경 전에 필요한 것

아래 중 하나면 먼저 계획을 남긴다.

1. 시트 스키마 변경
2. 저장 API 계약 변경
3. 삭제/아카이브 로직 변경
4. 다중 사용자나 동시성에 영향이 있는 변경
5. 운영 URL, 배포 설정, 권한 범위 변경

## 승인 없이는 하지 않는 것

1. 실사용 스프레드시트 구조 변경
2. 배포 URL 교체
3. 권한 범위 확대
4. 운영 데이터 삭제 방식 변경
