# RELIABILITY.md

## 목적

이 문서는 이 저장소에서 가장 자주 반복되는 안정성 검증 기준을 모아둔 문서다.

## 반드시 지켜야 할 안정성 기준

1. 첫 진입 시 화면이 비지 않아야 한다.
2. 저장 후 데이터가 로컬에 남아야 한다.
3. 저장 후 시트와 다시 맞춰 읽을 수 있어야 한다.
4. 오프라인과 온라인 상태가 사용자에게 드러나야 한다.
5. 충돌 가능성은 숨기지 말고 표기해야 한다.

## 최소 검증 체크리스트

1. 새로고침 후 상태 유지
2. 수동 동기화 동작
3. 실시간 동기화 상태 표기
4. 각 주요 탭 진입 가능 여부
5. 예약 생성, 수정, 삭제 흐름 회귀 여부
6. 서비스워커 캐시 영향 여부

## 작업 크기와 검증 원칙

안정성에 영향을 주는 작업은 크게 한 번에 밀지 않는다.

1. 큰 변경은 먼저 계획 문서로 나눈다.
2. 한 번의 변경은 한 가지 핵심 가설만 먼저 검증한다.
3. 증명에 실패하면 바로 멈춤 조건과 롤백 경계로 돌아간다.
4. 안정성 검증 결과는 `docs/continuity/current.md` 또는 관련 실행 계획에 남긴다.

## 현재 시스템에서 특히 취약한 지점

1. `index.html`에 집중된 로직
2. `localStorage`와 시트의 이중 상태
3. 부분 업데이트보다 전체 저장에 기대는 구조
