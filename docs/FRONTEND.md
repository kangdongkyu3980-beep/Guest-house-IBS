# FRONTEND.md

## 목적

이 문서는 프런트엔드 작업자가 `index.html` 중심 구조를 빠르게 이해하도록 돕기 위한 상위 안내 문서다.

## 현재 프런트엔드 특성

- 단일 HTML 파일에 UI, 상태, 렌더링, 저장, 동기화가 많이 모여 있다.
- 모바일 운영과 PWA 사용을 고려한 기능이 이미 들어가 있다.
- 기능 추가보다 경계 분리와 회귀 방지가 더 중요하다.

## 중요한 화면 축

- `개요`
- `식사`
- `룸맵`
- `IN/OUT`
- `달력`
- `게스트`
- `통계`

## 프런트엔드 작업 원칙

1. 새 로직이 들어가면 어느 탭과 어느 책임에 속하는지 설명 가능해야 한다.
2. 렌더 오류가 전체 화면 중단으로 이어지지 않게 해야 한다.
3. 모바일에서 버튼, 탭, 모달 흐름이 깨지지 않아야 한다.
4. 서비스워커와 PWA 흐름에 영향이 있으면 같이 검토해야 한다.

## Codex와 프런트엔드 작업할 때

1. 큰 UI 변경은 바로 구현하지 말고 먼저 영향 범위를 적는다.
2. 요청은 가능한 한 이슈처럼 쓴다.
3. 한 번에 화면 전체를 바꾸기보다 한 탭 또는 한 흐름 단위로 자른다.
4. 변경 뒤에는 모바일과 저장/재조회 흐름을 같이 확인한다.
