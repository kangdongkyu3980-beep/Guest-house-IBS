# SECURITY.md

## 목적

이 문서는 보안과 관련된 최소 기준을 정리한 상위 문서다.

## 기본 기준

1. 비밀값, 토큰, 장기 자격 증명은 가능한 한 코드에 직접 넣지 않는다.
2. 프런트엔드에 남는 값은 공개된다고 가정한다.
3. 외부 연동 URL, 권한 설정, 배포 주소는 문서화하되 민감 정보는 남기지 않는다.
4. 사용자 입력은 항상 출력 시 이스케이프를 고려한다.

## 현재 코드에서 주의할 점

1. 프런트엔드에 연동 URL이 남아 있을 수 있다.
2. Apps Script와 시트 권한 모델은 SaaS 기준 보안 모델이 아니다.
3. 현재 구조는 인증, 권한 분리, 감사 로그가 부족하다.

## 리뷰 전에 확인할 항목

1. 새 하드코딩 URL이나 자격 정보가 늘어나지 않았는가
2. 사용자 입력이 그대로 HTML로 삽입되지 않는가
3. 권한 모델이 필요한 변경인데 문서 없이 진행되지 않았는가
