## 🛠️ 개발 요청 처리 원칙

### 1. 코드 납품 기준 (필수)
- 항상 **완성된 전체 코드**를 제공할 것 (부분 코드 X)
- **복사 → VS Code 붙여넣기 → 바로 실행** 가능한 형태
- 코드 내 주석은 **한국어**로 작성
- 파일 저장 위치 및 실행 방법을 명확히 안내
- 오류 발생 가능 지점은 미리 경고하고 해결책 함께 제시

### 2. 설명 방식
- 전문 용어 사용 시 반드시 쉬운 말로 병행 설명
- "왜 이렇게 했는지" 이유를 간단히 덧붙일 것
- 단계가 많으면 번호를 매겨 순서대로 안내

### 3. 기술 선택 기준
- 가능하면 **기존 스택 유지** (HTML/JS 단일 파일 우선)
- 새 라이브러리 추가 시 CDN 링크 방식 선호 (npm 설치 최소화)
- Firebase 사용 시 → Firebase v9 modular SDK 기준
- 자동화는 Google Apps Script 또는 Make(구 Integromat) 우선

## 🔥 자주 쓰는 자동화 패턴

### WhatsApp 자동화
- WhatsApp Web + Google Apps Script 트리거
- 식수 리포트: 매일 특정 시간에 자동 전송
- 체크인/체크아웃 알림

### Google Sheets 연동
- Apps Script로 PMS 데이터 → Sheets 자동 기록
- 통계 데이터 Sheets로 백업

### Firebase 연동 (진행 중)
- Realtime Database 우선 (Firestore보다 단순)
- 인증: 이메일/비밀번호 or Anonymous
- 규칙: 인증된 사용자만 읽기/쓰기

## 📋 응답 형식 가이드

**기능 추가 요청 시**
1. 기능 요약 (1~2줄)
2. 어디에 추가하는지 (파일명, 위치)

1. 원인 파악 (쉬운 말로)

3. 기존 코드에서 어떤 부분을 교체하는지 명시

1. 플로우 다이어그램 (텍스트 or 마크다운)
2. 필요한 도구/계정
3. 단계별 구현 순서
